# 🛰️ R2 迁徙实录：数字游牧者的‘视觉家园’落成过程民族志

> **“在代码的荒原上，我们通过交换‘钥匙’，在云端为思想盖起了一座透明的灯塔。”**

## 📖 前言：为何需要“家园”？
在 2026 年的数字生存中，存储是一项具有政治意义的行为。我们面临的摩擦是：iPhone 生成的高质量 HEIC 照片（数 MB/张）如果直接存入本地 Git 仓库，会迅速导致同步堵塞。为了让“壮游”实录具备全球流转的轻盈感，我们开启了向 **Cloudflare R2** 的空间迁徙。

以下记录了这一“基建工程”作为人类学实验的全过程。

---

## 🏗️ 田野过程记录 (The Ethnographic Process)

### 1. 寻找“锚点”：Account ID 的定位
所有的迁徙第一步都是定位。我们通过 Cloudflare 的 Dashboard 抓取到了这组名为 `ACCOUNT_ID` 的 32 位标识符。它是我们在云端荒原中的“领地坐标”。
- **脱敏案例**：`R2_ACCOUNT_ID=<GATEWAY_ID_32_CHAR>`

### 2. 仪式感：与“唯一钥匙”的对峙
在创建 API Token 的过程中，我们进入了一个被称为“阅后即焚”的环节。
- **环节描述**：系统生成了一组 `Secret Access Key`。它在诞生后的 30 秒内如果未被人类捕获，将永远湮灭。
- **互喻火花**：momo 在本地创建了 `tcm-90-vault/core-secrets/r2.env`，这是一个物理级的隐私保护罩。这种“在阴影中存储光（Keys）”的行为，是数字主权意识的体现。
- **脱敏案例**：`R2_SECRET_ACCESS_KEY=<THE_INFITE_KEY_DO_NOT_SHARE>`

### 3. 炼金术：从转码到闪传
为了消灭格式带来的“认知余数”，我们编写了自动化管线。
- **动作**：把 `sips`（本地转码）与 `boto3`（云端上传）缝合在一起。
- **意义**：照片不再仅仅是本地的一个文件（File），它在被处理的一瞬间，分化为两个生命形态：
    - **肉身**：存在本地 `assets/images`。
    - **灵魂**：存在 R2 云端，通过一个全球唯一的 URL 完成“数字再生”。

---

## 🧠 媒介素养与认知迭代 (Reflections)

### 🧩 认知跃迁：从“静态拥有”到“动态访问”
在这一实验前，我们倾向于“把一切存在电脑里”。完成 R2 配置后，我们的心智模型拓宽为“按需访问”。我们不再拥有照片的物理实体，而是拥有了对该照片在云端位置的**“统治权”**。

### 🤝 人机协作：协议即信任
在处理“归档错误”时，我们没有进行情绪指责，而是通过共同通过一套 `Protocol`（协定）文件来对齐认知。
- **感悟**：在数字人类学实验室里，解决 Bug 的过程不是修复，而是**“制度建设”**。

## 🏹 对未来田野的预判
此系统落成后，我们在日本壮游调研中的采集能力将获得数倍提升。一张照片从快门按下到全球花园展示，延迟将缩短至 10 秒以内。这不仅是效率，它是对“当下”的一次精准狩猎。
